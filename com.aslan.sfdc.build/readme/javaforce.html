JavaForce -- Java Bindings to Salesforce

JavaForce is a java binding the to the Salesforce partner WSDL that hides much the SObject and governor limitations imposed by Salesforce. The JavaForce bindings have are the foundation of several open source projects:

  • SQLForce -- Jython module and a command line tool that add ANSI INSERT/UPDATE/DELTE/SELECT UNION/SELECT DISTINCT options to SOQL.
  • CopyForce -- Copy any Salesforce database to a local database (full or incremental; create local schema by introspecting Salesforce).

We have found that SQLForce and Jython is more than sufficient for managing Salesforce. However, when Jython is not enough, we use JavaForce jars directly. Examples:

  • Create a servlet that builds an Excel spreadsheet using the Apache/POI project.
  • Create a web service that will create an invoice in Oracle Financials from a Salesforce Opportunity

To get started:

  • Pick the JavaForce jar that is right for you.
    • The javaforceFull.jar contains JavaForce and all the required network/SOAP classes. Put javaforceFull.jar into your path and you can get started right away.
    • The javaforce.jar contains JavaForce and the code generated from the Salesforce partner WSDL. To use this jar you will need to include the following in your class path.
      • javax.xml.rpc
      • javax.wsdl
      • javax.xml.soap
      • org.apache.commons.logging
      • org.apache.commons.discovery
      • org.apache.axis
      • org.apache.log4j
      • javax.mail
      • javax.activation
  • Look at the sample code.
  • Look at the javadoc starting with the package com.aslan.sfdc.partner.
